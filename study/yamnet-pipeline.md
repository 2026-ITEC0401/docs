# YAMNet 2단계 분류 파이프라인 정리

## 전체 흐름

```
마이크 녹음 (1초)
    ↓
YAMNet (1차)
    ├── 521개 카테고리 중 하나로 판별 (1차 필터링)
    └── 1024차원 벡터 생성 (특징 추출)
    ↓
직접 매핑? → 바로 알림 (개 짖음, 물 소리, 초인종, 창문 깨지는 소리)
2차 분류 대상? → 1024차원 벡터를 2차 분류기로 전달
그 외? → 무시 (대화, 음악 등)
    ↓
2차 분류기: 1024 → 256 → 128 → 4 (노크/도어락/사이렌/아기울음)
    ↓
confidence 70% 이상이면 서버로 알림 전송
```

## 521개 카테고리 vs 1024차원 벡터

- **521개**: YAMNet이 분류할 수 있는 소리 종류 (최종 출력)
- **1024차원**: 소리의 특징을 숫자로 표현한 벡터 (중간 과정)

이 두 가지는 별개. 1024차원 벡터를 기반으로 521개 중 하나를 판별하는 구조.
우리는 521개 분류 결과(1차 필터링)와 1024차원 벡터(2차 분류기 입력)를 동시에 사용.

## 2차 분류기의 차원 축소 과정

```
1024차원 벡터 (YAMNet에서 받은 입력)
    ↓
Dense(256) - 1024개 특징을 조합해서 새로운 256개 특징 생성
    ↓
Dropout(0.3) - 30% 랜덤 제거 (과적합 방지)
    ↓
Dense(128) - 256개를 조합해서 128개로 압축
    ↓
Dropout(0.3)
    ↓
Dense(4, softmax) - 128개에서 4개 카테고리 확률 출력
    → 예: [노크 98%, 도어락 1%, 사이렌 0.5%, 아기울음 0.5%]
```

### 핵심 포인트
- "필요없는 데이터를 제거"하는 게 아니라 "필요한 특징을 조합·압축"하는 것
- 마지막 Dense의 숫자(4)는 우리 카테고리 수에 따라 결정됨
- 카테고리를 늘리면 Dense(5), Dense(10) 등으로 자동 변경

## 왜 적은 데이터(90개)로도 학습이 가능한가? (Transfer Learning)

- YAMNet이 이미 수백만 개의 AudioSet 데이터로 "소리 특징 추출 능력"을 학습해둠
- 우리의 2차 분류기는 원시 오디오가 아니라 이미 정제된 1024차원 벡터를 입력으로 받음
- 그래서 Dense 3개(약 26만 개 파라미터)만 학습하면 되고, 과적합 위험이 크게 줄어듦

### 비유
- **from scratch**: 한글도 모르는 사람에게 책 3권 주고 시험 보라는 것
- **Transfer Learning**: 이미 한글을 마스터한 사람에게 책 3권 주고 특정 분야 시험 보라는 것

## 학습 vs 추론

| 구분 | 어디서 | 뭘 하는지 | 시간 |
|------|--------|----------|------|
| 학습 (Training) | Colab | 26만 개 가중치의 최적값 찾기 | 수분 |
| 추론 (Inference) | 라즈베리파이 | 저장된 가중치로 계산만 수행 | 0.05~0.1초 |

- tflite 파일에는 학습 완료된 가중치가 저장되어 있음
- 라즈베리파이에서는 곱셈/덧셈만 하므로 빠름

## Threshold 설정

```python
YAMNET_THRESHOLD = 0.2      # 1차 직접 매핑용
CLASSIFIER_THRESHOLD = 0.7  # 2차 분류기용
```

- YAMNet 점수는 softmax가 아니라 0.3~0.4도 강한 신호 → 0.2로 낮게 설정
- 2차 분류기는 softmax 출력이라 0.7로 설정
