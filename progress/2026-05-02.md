# 2026-05-02 진행 내역

## 1. YAMNet 프리필터 구조 변경

### 사이렌 → 2차 분류기로 이동
- 기존: YAMNet 직접 매핑 8개 (사이렌 포함)
- 변경: 직접 매핑 7개, 사이렌은 2차 분류기로 이동
- 이유: 한국 사이렌은 서양과 패턴이 달라 분류기가 판단해야 함
- 관련 파일: `raspberry-pi/app.py`, `docs/problem/yamnet-sound-classification-flow.md`

### 카테고리 리팩토링 (소리 테스트 결과 반영)
- **초인종/인터폰 통합**: 초인종 → 인터폰으로 합침 (YAMNet이 둘 다 Telephone로 분류)
- **벨소리 제거**: YAMNet이 Music(132)으로 분류해서 1차에서 차단됨, 한국에서도 벨소리를 음악으로 설정
- **아기울음 → 2차 분류기로 이동**: 사이렌과 혼동 방지 (YAMNet이 사이렌을 Crying으로 분류하는 문제)
- **threshold 0.5 → 0.2**: YAMNet 점수는 softmax가 아니라 0.3~0.4도 강한 신호

### 현재 구조 요약
- 직접 매핑 (5개): 개 짖음, 물 소리, 인터폰(구 초인종), 노크 소리, 창문 깨지는 소리
- 2차 분류기 (5개): 도어락, 인터폰, 가전제품, 사이렌, 아기 울음
- 제거 (1개): 휴대폰 벨소리

## 4. 프론트엔드-백엔드 연동 분석

### 백엔드 API (FastAPI) - 준비 완료
- `POST /api/alerts` - 라즈베리파이 → 서버
- `POST /api/heartbeat` - 라즈베리파이 연결 확인
- `GET /api/dashboard` - 프론트엔드 대시보드 데이터
- `PATCH /api/alerts/{id}/read` - 알림 읽음 처리
- `WS /ws` - 프론트엔드 실시간 알림 (WebSocket)

### 프론트엔드 - 연동 미완료
- 현재 `IS_MOCK_MODE = true` (목업 데이터만 사용)
- 프론트엔드 담당자가 수정해야 할 것:
  1. `IS_MOCK_MODE = false`로 변경
  2. WebSocket 연결: `ws://서버IP:8000/ws`
  3. Dashboard API 호출: `GET http://서버IP:8000/api/dashboard`
  4. 서버 주소를 변수로 관리 (`SERVER_URL`, `WS_URL`)

### 백엔드 config.py 주의사항
- 아직 기존 11개 카테고리 그대로 유지
- 테스트 후 초인종/인터폰 통합, 벨소리 제거 반영 필요

```bash

source ~/hearo-env/bin/activate

# 2. 최신 코드 받기
cd ~/Hearo/raspberry-pi
git pull

# 3. 실행
python app.py

# 4. 종료
Ctrl + C

# 5. 라즈베리파이 끄기 (전원 뽑기 전 필수)
sudo shutdown -h now
```
